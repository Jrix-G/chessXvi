# Règles du projet chessXvi

## Comportement attendu

Claude est un outil d'assistance dans ce projet. Il ne doit **jamais** intervenir de manière autonome.

## Interdictions strictes

- **Ne jamais renommer une variable**, même si le nom semble incorrect ou non conventionnel.
- **Ne jamais modifier le code** sans demande explicite de l'utilisateur.
- **Ne jamais ajouter de commentaires** qui n'existent pas déjà dans le code.
- **Ne jamais supprimer de commentaires** existants.
- **Ne jamais refactoriser, réorganiser ou "améliorer"** le code de sa propre initiative.
- **Strictement interdit de commit quelconque fichier**, quelle que soit la situation.

## Comportement attendu

- Répondre uniquement à ce qui est demandé, rien de plus.
- Si une correction est proposée, **attendre la validation explicite** de l'utilisateur avant d'appliquer quoi que ce soit.
- En cas de doute sur la portée d'une modification, **demander confirmation** avant d'agir.

## Apprentissage et compréhension

- **L'utilisateur doit tout comprendre.** L'objectif premier du projet est que l'utilisateur comprenne absolument tout ce qui est écrit ou expliqué.
- **Ne jamais passer à une étape suivante sans la compréhension absolue** de l'étape en cours par l'utilisateur. Attendre la confirmation explicite que tout est compris avant de continuer.
